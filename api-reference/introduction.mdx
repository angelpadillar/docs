# API Reference

Welcome to the comprehensive Buena.ai API v2 reference. This section provides detailed documentation for every endpoint, parameter, and response format.

## Base URL

All API requests are made to:

```
https://api.buena.ai/api/v2
```

## Authentication

Every request requires an API key in the header:

```bash
curl -H "x-api-key: YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     "https://api.buena.ai/api/v2/health"
```

<Note>
  Need an API key? Create one through your [dashboard](https://app.buena.ai) or
  see our [authentication guide](/authentication).
</Note>

## Quick Reference

### Most Used Endpoints

| Endpoint                           | Method | Purpose                  | Permission          |
| ---------------------------------- | ------ | ------------------------ | ------------------- |
| `/health`                          | GET    | Health check             | None                |
| `/keys`                            | POST   | Create API key           | None                |
| `/linkedin/scheduleLinkedInAction` | POST   | Schedule LinkedIn action | `linkedin:schedule` |
| `/linkedin/uploadProspects`        | POST   | Upload prospects         | `linkedin:upload`   |
| `/leads`                           | GET    | List leads               | `leads:read`        |
| `/leads`                           | POST   | Create lead              | `leads:write`       |
| `/enrich`                          | POST   | Enrich leads             | `leads:enrich`      |

### Response Format

All successful responses follow this format:

```json
{
  "success": true,
  "data": {
    // Response data
  },
  "message": "Optional success message"
}
```

### Error Format

All errors return:

```json
{
  "error": true,
  "code": "ERROR_CODE",
  "message": "Human-readable description",
  "version": "2.0",
  "timestamp": "2024-01-20T15:30:00Z"
}
```

## Permission System

API keys require specific permissions for different endpoints:

<AccordionGroup>
  <Accordion title="LinkedIn Permissions">
    - `linkedin:schedule` - Schedule LinkedIn actions - `linkedin:upload` -
    Upload prospect lists - `linkedin:read` - Read LinkedIn data
  </Accordion>
  <Accordion title="Lead Permissions">
    - `leads:read` - Read lead data - `leads:write` - Create/update leads -
    `leads:enrich` - Enrich lead data
  </Accordion>
  <Accordion title="User Permissions">
    - `users:read` - Read user data - `admin` - Administrative access
  </Accordion>
</AccordionGroup>

### Example Request

Here's a quick example to get you started:

<CodeGroup>

```bash cURL
curl -X GET "https://api.buena.ai/api/v2/health" \
  -H "x-api-key: YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch("https://api.buena.ai/api/v2/health", {
  headers: {
    "x-api-key": "YOUR_API_KEY",
  },
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

response = requests.get(
    'https://api.buena.ai/api/v2/health',
    headers={'x-api-key': 'YOUR_API_KEY'}
)

print(response.json())
```

</CodeGroup>
