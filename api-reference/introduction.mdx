# API Reference

Welcome to the Buena.ai API v2 reference documentation. This section provides detailed documentation for every endpoint, parameter, and response format.

## Base URL

All API requests are made to:

```
https://api.buena.ai/api/v2
```

## Authentication

Every request requires an API key in the header:

```bash
curl -H "x-api-key: YOUR_API_KEY" \
     -H "Content-Type: application/json" \
     "https://api.buena.ai/api/v2/health"
```

<Note>
Need an API key? Create one through your [dashboard](https://app.buena.ai) or see our [authentication guide](/authentication).
</Note>

## Quick Start

Test your API key with a simple health check:

```bash
curl -X GET "https://api.buena.ai/api/v2/health" \
  -H "x-api-key: YOUR_API_KEY"
```

## Response Format

All successful responses follow this format:

```json
{
  "success": true,
  "data": {
    // Response data here
  },
  "message": "Optional success message"
}
```

## Error Format

All errors return:

```json
{
  "error": true,
  "code": "ERROR_CODE",
  "message": "Human-readable description",
  "version": "2.0",
  "timestamp": "2024-01-20T15:30:00Z"
}
```
